# Security

## Reporting Security Issues

Found a security vulnerability? Please email me directly instead of opening a public issue.

**Email:** your.email@example.com

I'll respond within 48 hours and work on fixing it as soon as possible.

## What to Include

When reporting a security issue, please provide:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## Security Features

This project includes:
- Password hashing with bcrypt
- Prepared statements for SQL queries
- CSRF token protection
- XSS prevention
- Secure session management
- File upload validation
- Role-based access control

## Best Practices for Users

1. Change default passwords immediately
2. Use strong passwords (12+ characters)
3. Keep PHP and MySQL updated
4. Use HTTPS in production
5. Set proper file permissions
6. Backup your database regularly

## For Production

Before going live:
- Enable HTTPS
- Change all default passwords
- Set `display_errors = Off`
- Review file permissions
- Test security features
- Keep software updated

## Supported Versions

Currently supporting version 1.0.x with security updates.

Thanks for helping keep this project secure!
