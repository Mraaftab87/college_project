# Contributing

Thanks for your interest in contributing! Here's how you can help make this project better.

## How to Contribute

### Reporting Bugs

Before creating a bug report, check if the issue already exists. When reporting:

- Describe what happened and what you expected
- Include steps to reproduce the bug
- Add screenshots if helpful
- Mention your environment (PHP version, OS, browser)

### Suggesting Features

Have an idea? Open an issue and describe:

- What problem it solves
- How it would work
- Why it would be useful

### Pull Requests

1. Fork the repo
2. Create a branch for your changes
3. Make your changes
4. Test everything works
5. Commit with a clear message
6. Push and open a PR

## Code Style

- Follow existing code patterns
- Use clear variable names
- Add comments for complex logic
- Use prepared statements for database queries
- Validate all user inputs

## Commit Messages

Keep them simple and clear:
```
Add bulk delete feature
Fix login session timeout
Update product search logic
```

## Testing

Before submitting:
- Test your changes thoroughly
- Check on different browsers
- Try different user roles
- Make sure nothing breaks

## Questions?

Open an issue with the "question" label if you need help.

Thanks for contributing!
